# Vendor knowledge

This directory holds the per-vendor capability files the cross-vendor
machinery depends on. Each vendor knowledge file is a volatility-tagged
snapshot of one CLI agent's capabilities — canonical instructions file,
skills primitive, subagents, hooks, auth, rate limits, cost, MCP
support, headless invocation.

## What ships

| File | Vendor | What it covers |
|---|---|---|
| [`vendor-knowledge/claude-code.md`](vendor-knowledge/claude-code.md) | Anthropic — Claude Code | Native `CLAUDE.md` lookup, `.claude/skills/`, hook events, rate limits, MCP client + server, headless invocation. |
| [`vendor-knowledge/codex-cli.md`](vendor-knowledge/codex-cli.md) | OpenAI — Codex CLI | Native `AGENTS.md` lookup, `.agents/skills/` skills primitive (open Agent Skills standard), `.codex/agents/` subagents, hooks, rate limits, MCP client + server, `codex exec` headless. |
| [`vendor-knowledge/antigravity-cli.md`](vendor-knowledge/antigravity-cli.md) | Google — Antigravity CLI (coding CLI; config root inherits from `~/.gemini/`) | Reads `AGENTS.md` via `.gemini/settings.json` override, skills primitive, settings.json options, `agy --print` headless. |

## Also in this directory

| File | What it covers |
|---|---|
| [`model-capabilities.md`](model-capabilities.md) | Per-model selection + cost reference. Distinct from CLI knowledge — covers the model itself, not the harness around it. |
| [`skills-index.md`](skills-index.md) | Auto-generated index of authored skills. Regenerated by `scripts/generate-skill-index.sh` on every commit. |
| [`refresh-log.md`](refresh-log.md) | Append-only ledger of `/refresh-vendor` walks. |

## Refresh cadence

Every claim in every file carries a volatility tag (`[STABLE]` /
`[MEDIUM]` / `[VOLATILE]`) and a citation. The [`refresh-log.md`](refresh-log.md)
is the append-only ledger of when each file was last walked.

`/refresh-vendor <vendor>` walks the per-claim drift detection per
[`markdowns/protocols/refresh-vendor.md`](../protocols/refresh-vendor.md).
